# Bicubic Kernel LUT

## Change

Added a precomputed lookup table for `bicubic()` (Catmull-Rom kernel), following
the same pattern as `lanczos3LUT`:

- 1024-entry table over [0, 2) — the kernel's support radius
- Symmetric kernel, only positive half stored
- Linear interpolation between table entries
- Replaces polynomial evaluation (`1.5x³ - 2.5x² + 1` / `-0.5x³ + 2.5x² - 4x + 2`)

## Files changed

- `internal/tile/resample.go` — Added `bicubicLUTSize`, `bicubicTable`, init, and
  `bicubicLUT()`. Replaced `bicubic()` calls in both bicubic resampling loops with
  `bicubicLUT()`.

## Motivation

Profiling showed ~3.25s cumulative CPU in bicubic polynomial evaluation. While cheaper
than Lanczos sin() calls, the LUT eliminates this entirely with a single table lookup
+ lerp per evaluation.
